# 📖 About This Repository

This repository is a compact preparation guide for the oral admission interview of the **Computer Science MSc program** at the **University of Milan (UNIMI)**.

The goal is **not** to replace full university courses.
Instead, it helps you:

* Quickly review the most important CS fundamentals
* Practice short oral explanations
* Learn common interview topics
* Improve confidence before the interview
* Identify weak areas for quick revision

This guide is especially useful if:

* You come from a different academic background
* You need a fast refresher
* You want concise interview-oriented explanations

---

# 🎯 What the Interview Usually Tests

Based on student experiences and common topic lists, the interview is usually a short oral evaluation of your foundations in computer science.

Typical questions may include:

* A simple programming or algorithm task
* A database or SQL question
* An operating systems or networking concept
* A computer architecture topic
* A basic mathematics or statistics question

The questions are generally:

* Fundamental
* Broad
* Conceptual
* Explanation-focused rather than deeply theoretical

---

# 🧠 Best Structure for Answering Questions

A good interview answer usually follows this structure:

## 1️⃣ Definition

Start with a clear definition.

## 2️⃣ Main Idea

Explain the core concept in simple words.

## 3️⃣ Small Example

Give a quick real-world or technical example.

## 4️⃣ Tradeoff / Complexity / Use Case

Mention:

* Time complexity
* Performance tradeoff
* Typical usage
* Advantages/disadvantages

---

# 🛟 If You Forget Something During the Interview

It is completely normal to forget details under stress.

Instead of staying silent, try phrases like:

```text id="interview-phrases"
"I do not remember the exact formal definition, but conceptually it means ..."
```

```text id="interview-phrases-2"
"Let me reason about it step by step."
```

```text id="interview-phrases-3"
"The main idea is ..., and a typical example is ..."
```

```text id="interview-phrases-4"
"I may not remember every implementation detail, but the tradeoff is ..."
```

Interviewers usually care more about:

* Reasoning ability
* Understanding
* Communication clarity

than memorizing exact textbook definitions.
